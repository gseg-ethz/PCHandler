## [2.0.0](https://github.com/gseg-ethz/PCHandler/compare/v1.0.1...v2.0.0) (2026-06-12)


### ⚠ BREAKING CHANGES

* **05-04:** CsvHandler.load now raises ValueError when scalar_fields names are not found in the file header. Previously, missing names silently fell back to positional column mapping, which could produce column-mis-mapped data without warning. Any caller passing scalar_fields=["name"] where "name" is absent from the parsed header will now get an actionable ValueError:   CsvHandler.load: requested scalar fields not in header: ['name'].   Available: [...]. Pass an explicit `column_names_row` if the header lives outside line -1.
* general library structure
* Added and initalized v2
* Changed structure of filters

* **RTD version-label override:** `docs/source/conf.py` shipped with `version = "1.0.1"` at the
  v2.0.0 release tag because `release-please-config.json:26` had `extra-files: ["docs/conf.py"]`
  pointing at a non-existent path (the live docs source lives at `docs/source/conf.py`). The
  release-please bot therefore did not stamp the file, leaving the previous `1.0.1` value in
